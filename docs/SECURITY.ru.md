# Безопасность

## Принципы проектирования

Transmitter разработан для **развёртывания в домашней сети**, где все пользователи являются доверенными. Функции безопасности реализованы для предотвращения случайного раскрытия данных и защиты учётных данных.

## Аутентификация и авторизация

- **Веб-интерфейс:** без аутентификации пользователей. Доступен только в домашней сети. Если впоследствии потребуется внешний доступ, добавьте middleware с Basic Auth или сессионными куки.
- **Telegram-бот:** белый список по ID пользователя. Неавторизованные сообщения игнорируются (без ответа, логируется на уровне warn).

## Управление учётными данными

- **Transmission RPC:** учётные данные хранятся в файле `.env` и переменных окружения контейнера. Никогда не передаются на фронтенд — проксируются на стороне сервера в Go.
- Учётные данные передаются через заголовок `Authorization: Basic <base64>` (HTTP Basic Auth)
- Переменные окружения загружаются через `godotenv`, не отображаются в логах

## Защита от CSRF

- **Transmission RPC:** использует собственный механизм CSRF (заголовок `X-Transmission-Session-Id`)
- Go-прокси управляет жизненным циклом токена прозрачно:
  1. Получает session ID при первом запросе (таймаут 10 сек)
  2. При ответе 409 Conflict извлекает новый ID из заголовка
  3. Повторяет запрос с новым токеном (одна повторная попытка)
  4. Кэширует действующий токен в памяти (`atomic.Value` с singleflight)

## Сетевая безопасность

- **CORS:** явная проверка источника (без wildcards), защита от DNS rebinding атак
- `network_mode: host` в Docker — упрощает доступ к Transmission через localhost
- Нет внешнего доступа — предполагается частная сеть

## Валидация запросов

- **Белый список RPC:** разрешены только 6 методов:
  - `torrent-get`, `torrent-add`, `torrent-start`, `torrent-stop`, `torrent-remove`, `session-get`
- **Ограничение размера запроса:** максимум 1 МБ (защита от DoS)
- Прямая передача JSON-RPC (пользовательский ввод в RPC-вызовах только в данных торрента)

## Рекомендации для внешнего доступа

При открытии Transmitter в интернет или недоверенные сети:

1. **Добавьте аутентификацию** — Basic Auth или session middleware в Go
2. **Используйте HTTPS** — обратный прокси с TLS (nginx, Caddy и т.д.)
3. **Ограничьте диапазон IP** — правила в firewall или обратном прокси
4. **VPN/туннель** — SSH port forwarding или WireGuard для удалённого доступа
5. **Rate limiting** — реализуйте ограничение запросов на эндпоинте `/api/rpc`
6. **Смените настройки по умолчанию** — используйте надёжные учётные данные Transmission

## Конфиденциальность данных

- Все данные Transmission (торрент-файлы, учётные данные) обрабатываются на стороне сервера
- Фронтенд получает только: метаданные торрентов, статус, прогресс
- Сессионные токены не передаются клиенту
- Взаимодействия с Telegram логируются на настроенном уровне
